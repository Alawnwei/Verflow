# AI驱动测试自动化平台 - 概要设计文档

## 1. 项目概述

### 1.1 项目背景
本项目旨在构建一个基于AI的测试自动化平台，实现从需求文档到测试执行的全流程自动化。平台利用大语言模型（LLM）技术，自动解析需求文档、生成测试用例、创建自动化脚本，并支持测试执行和结果分析。

### 1.2 项目目标
- **自动化测试用例生成**：基于需求文档自动生成高质量测试用例
- **智能脚本生成**：根据测试用例自动生成可执行的测试脚本
- **多源需求支持**：支持BSD文档、JIRA、Lark等多种需求来源
- **AI自我修复**：测试失败时自动尝试修复脚本
- **完整测试编排**：支持冒烟测试、E2E测试、回归测试

### 1.3 核心价值
| 价值点 | 描述 |
|-------|------|
| 效率提升 | 减少80%的测试用例编写时间 |
| 质量保证 | AI预审核确保用例质量 |
| 持续改进 | 测试结果反馈驱动持续优化 |
| 知识沉淀 | 测试资产结构化存储和复用 |

## 2. 整体架构设计

### 2.1 架构风格
采用分层架构模式，包含表现层、业务逻辑层、数据访问层和基础设施层。

### 2.2 模块划分

| 模块 | 职责 | 核心功能 |
|-----|------|---------|
| **需求解析模块** | 解析多源需求文档 | 文档解析、AI分析、结构化输出 |
| **用例生成模块** | 生成和管理测试用例 | AI生成、审核流程、用例管理 |
| **脚本生成模块** | 生成自动化测试脚本 | 代码生成、人工编辑、版本管理 |
| **测试编排模块** | 编排和执行测试 | 测试套件管理、并行执行、调度 |
| **执行引擎模块** | 执行测试脚本 | 脚本执行、结果收集、AI修复 |
| **报告分析模块** | 生成和展示报告 | 测试报告、趋势分析、通知推送 |
| **MCP集成模块** | 外部工具集成 | 工具调用、上下文管理 |

## 3. 核心业务流程

### 3.1 需求导入与解析流程

```mermaid
flowchart TD
    A[用户上传需求文档] --> B{判断文档类型}
    B -->|BSD文档| C[文档解析器]
    B -->|JIRA| D[JIRA API调用]
    B -->|Lark| E[Lark API调用]
    B -->|截图| F[OCR识别]
    B -->|设计稿| G[图像分析]
    C --> H[统一数据模型]
    D --> H
    E --> H
    F --> H
    G --> H
    H --> I[AI语义分析]
    I --> J[提取功能点]
    J --> K[识别业务规则]
    K --> L[输出结构化数据]
```

### 3.2 测试用例生成流程

```mermaid
flowchart TD
    A[解析结果输入] --> B[AI生成用例]
    B --> C[用例模板匹配]
    C --> D[参数化填充]
    D --> E[优先级排序]
    E --> F[用例去重]
    F --> G[AI预审核]
    G --> H{审核通过?}
    H -->|是| I[人工Review]
    H -->|否| J[自动优化建议]
    J --> K[用户确认修改]
    K --> B
    I --> L{Review通过?}
    L -->|是| M[用例冻结]
    L -->|否| N[修改意见反馈]
    N --> B
```

### 3.3 脚本生成与执行流程

```mermaid
flowchart TD
    A[已审核用例] --> B[选择测试框架]
    B --> C[AI代码生成]
    C --> D[参数注入]
    D --> E[断言生成]
    E --> F[脚本优化]
    F --> G{需要人工编辑?}
    G -->|是| H[人工编辑]
    G -->|否| I[脚本冻结]
    H --> J[代码审查]
    J --> K{审查通过?}
    K -->|是| I
    K -->|否| H
    I --> L[执行测试]
    L --> M{测试失败?}
    M -->|是| N[AI自我修复]
    N --> O{修复成功?}
    O -->|是| P[更新脚本]
    O -->|否| Q[创建工单]
    P --> L
    M -->|否| R[生成报告]
    Q --> R
```

## 4. 关键技术选型

### 4.1 技术栈

| 分类 | 技术 | 版本 | 选型理由 |
|-----|------|------|---------|
| **前端框架** | React | 18.x | 成熟稳定，生态丰富 |
| **后端框架** | Spring Boot | 3.x | Java生态，企业级支持 |
| **数据库** | PostgreSQL | 15.x | 支持JSON字段，全文搜索 |
| **缓存** | Redis | 7.x | 高性能缓存，支持分布式锁 |
| **消息队列** | RabbitMQ | 3.x | 可靠消息传递，支持延迟队列 |
| **LLM服务** | DeepSeek-v4-flash | - | 高性能，支持多模型fallback |
| **测试框架** | Playwright | 1.x | 现代Web自动化，支持多浏览器 |
| **CI/CD** | GitLab CI | - | 与GitLab深度集成 |
