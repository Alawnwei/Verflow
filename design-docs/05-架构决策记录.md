## ADR-001: 选择PostgreSQL作为数据库管理系统

### 状态
**已接受**

### 背景
需要选择一个稳定、可靠的数据库管理系统来存储测试用例、脚本、执行记录等数据。

### 决策
选择 **PostgreSQL 15.x** 作为数据库管理系统。

### 理由

| 因素 | PostgreSQL | MySQL | MongoDB |
|-----|-----------|-------|---------|
| JSON支持 | 原生支持JSONB | 5.7+支持JSON | 文档型数据库 |
| 全文搜索 | 原生支持 | 需要插件 | 需要插件 |
| 事务支持 | 强一致性 | 强一致性 | 最终一致性 |
| 扩展性 | 良好 | 良好 | 优秀 |
| 社区支持 | 活跃 | 非常活跃 | 活跃 |
| 企业级特性 | 支持 | 支持 | 有限 |

### 影响
- 需要学习PostgreSQL的JSONB查询语法
- 备份和恢复策略需要适配PostgreSQL

## ADR-010: 不支持移动端测试框架

### 状态
**已接受**

### 背景
根据业务需求分析，公司业务主要集中在Web端（2C和2B），移动端测试需求较少。

### 决策
**不支持移动端测试框架**（如Appium），专注于Web端测试。

### 理由

| 因素 | 支持移动端 | 不支持移动端 |
|-----|-----------|-------------|
| 业务需求 | 2C/2B主要在Web端 | 符合当前业务现状 |
| 开发成本 | 高 | 低 |
| 维护成本 | 高 | 低 |
| 学习曲线 | 额外学习成本 | 专注Web技术栈 |
| 资源投入 | 需要额外设备/模拟器 | 现有资源足够 |

### 影响
- 如果未来需要移动端测试，需要重新评估
- 文档和培训材料不需要包含移动端内容
